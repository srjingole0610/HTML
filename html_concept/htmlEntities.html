Understanding HTML Entities

What are HTML Entities?

HTML entities are special characters that are not part of the standard ASCII character set or characters that have special meaning in HTML. They are used to:

  • Display reserved characters that have special meaning in HTML (like < and >)
  • Show characters that are difficult to type on a keyboard
  • Represent symbols and special characters across different character encodings

Entity Syntax

HTML entities can be written in three formats:

  1. Named entity: &entityname;
  2. Decimal reference: &#decimalnumber;
  3. Hexadecimal reference: &#xhexnumber;

Common HTML Entities

Character Entity Name Decimal Hexadecimal Description
< &lt; &#60; &#x3C; Less than
> &gt; &#62; &#x3E; Greater than
& &amp; &#38; &#x26; Ampersand
© &copy; &#169; &#xA9; Copyright
® &reg; &#174; &#xAE; Registered trademark

Special Characters and Symbols

Currency Symbols

€ - Euro (&euro;)

£ - Pound (&pound;)

¢ - Cent (&cent;)

¥ - Yen (&yen;)

Mathematical Symbols

× - Multiplication (&times;)

÷ - Division (&divide;)

± - Plus-minus (&plusmn;)

≠ - Not equal (&ne;)

Best Practices

  • Always use HTML entities for special characters to ensure proper rendering across different browsers and character encodings
  • Named entities are more readable and maintainable than numeric references
  • Use UTF-8 encoding in your HTML documents to support a wider range of characters directly
  • Remember to encode special characters in attributes as well as content
