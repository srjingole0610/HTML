HTML Void Elements (Empty Elements)

1. Line Break Element <br>

This is a line
break example

Usage: Forces a line break. No attributes required.

2. Horizontal Rule <hr>

Text above the line

Text below the line

Attributes: size, width, color, align

3. Image Element <img>

Required Attributes: src, alt

Optional Attributes: width, height, loading, srcset, sizes

4. Input Elements <input>

Common Types: text, password, checkbox, radio, submit, reset, button, date, email, file, number

Common Attributes: type, name, value, placeholder, required, disabled, readonly

5. Meta Tags <meta>

Common Attributes: name, content, charset, http-equiv

Common Uses: character encoding, viewport settings, SEO descriptions, author info

6. Link Element <link>

Example: <link rel="stylesheet" href="styles.css">

Required Attributes: rel, href

Common Uses: CSS stylesheets, favicons, preloading resources

7. Area Element <area>

Required Attributes: shape, coords, alt

Shapes: rect, circle, poly

8. Base Element <base>

Example: <base href="https://www.example.com/">

Attributes: href, target

Note: Must be in head section, only one per document

9. Embed Element <embed>

Required Attributes: src

Optional Attributes: type, width, height

10. Source Element <source>

Required Attributes: src

Optional Attributes: type, media, sizes, srcset

11. Track Element <track>

Required Attributes: src, kind, srclang

Optional Attributes: label, default

12. Wbr Element <wbr>

This is a very long word: supercalifragilisticexpialidocious

Usage: Indicates a word break opportunity. No attributes required.

13. Col Element <col>

Column 1 Column 2

Attributes: span, style, class

Usage: Defines properties for table columns

14. Param Element <param>

Required Attributes: name, value

Usage: Defines parameters for plugins used in object element

Key Points About Void Elements:

  • Void elements have no content and no closing tag
  • They are also called empty elements or self-closing tags
  • They can have attributes but cannot have content inside
  • Both <tag> and <tag/> syntax are valid in HTML5
  • Most void elements require specific attributes to function properly
  • Void elements are essential for various functionalities like media embedding, form inputs, and document metadata
