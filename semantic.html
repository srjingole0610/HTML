Deep Dive into Semantic Elements

Structural Semantic Elements

<header>
Represents introductory content, typically containing navigation and branding elements. Can be used multiple times in a document for different sections.
<nav>
Defines a section of navigation links. Best used for major navigation blocks. Improves accessibility by helping screen readers identify navigation sections.
<main>
Specifies the main content area of a document. Should be unique to the document and exclude content repeated across documents like sidebars and navigation.
<article>
Represents a self-contained composition that can be independently distributable or reusable. Perfect for blog posts, news articles, or forum posts.
<section>
Defines a thematic grouping of content, typically with a heading. Helps create a logical document structure.
<aside>
Contains content tangentially related to the content around it. Commonly used for sidebars or call-out boxes.
<footer>
Represents a footer for its nearest sectioning content or sectioning root element. Can contain author info, copyright data, related links, etc.

Text-level Semantic Elements

<time>
Represents time and dates. The datetime attribute helps machines understand the time format.
<mark>
Indicates text of specific relevance or importance in a context.
<em>
Indicates emphasis with semantic importance, not just italic styling.
<strong>
Represents strong importance, seriousness, or urgency.
<cite>
Represents a reference to a creative work.

Semantic Attributes

role
Defines the purpose of an element in terms of ARIA landmarks, helping with accessibility.
aria-label
Provides a text label for elements that may not have visible text, crucial for screen readers.
lang
Specifies the language of an element's content, important for accessibility and SEO.
